# Contributing

Thank you for your interest in this project.

This is an early-stage learning repository focused on open-source maintainer workflows.

## Good first contributions

- Improve README clarity
- Add examples to documentation
- Improve issue triage checklist
- Improve pull request review checklist
- Add small tests for existing behavior

## Contribution process

1. Open an issue describing the improvement.
2. Make a small focused change.
3. Open a pull request.
4. Explain what changed and why.
5. Wait for review before merging.

## Style

- Keep explanations beginner-friendly.
- Prefer small changes over large rewrites.
- Add tests when changing behavior.
- Update documentation when changing user-facing behavior.
